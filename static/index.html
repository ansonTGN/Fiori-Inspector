Guía para empezar desde cero

Usa Fiori Inspector Studio sin saber SAPUI5, DOM ni Rust

Esta herramienta mira una pantalla Fiori como si fuera un mapa técnico: identifica botones, campos, tablas, textos, bindings de datos, endpoints OData y selectores seguros para preparar automatizaciones profesionales.

1

Arranca la aplicación

Desde la carpeta del proyecto ejecuta el servidor local. Luego abre la URL que aparece en pantalla.

cargo run -- --config config/local.toml serve
2

Prepara Chrome DevTools Protocol

Solo es necesario para analizar una sesión Fiori viva. Normalmente la app lo lanza sola; este comando sirve si quieres abrirlo manualmente.

google-chrome --remote-debugging-port=9222 --user-data-dir=./.browser-profile-cdp
3

Captura o pega HTML

Usa “Capturar sesión viva” para Fiori real, o “Analizar HTML” para aprender sin conectarte a SAP.

Conceptos básicos

Qué significa cada cosa

FioriLa interfaz web moderna de SAP. Es la pantalla que ve el usuario.
SAPUI5 / UI5La librería JavaScript que construye los controles: botones, tablas, formularios y vistas.
DOMEl HTML real que el navegador renderiza. Sirve para buscar selectores, pero no explica toda la lógica UI5.
Control UI5Objeto lógico de SAPUI5. Es más estable que mirar solo etiquetas HTML.
BindingRelación entre un campo de pantalla y un dato del modelo. Ayuda a entender qué información se muestra.
ODataServicios de datos que suelen alimentar Fiori. Permiten descubrir entidades, filtros y llamadas reales.
SelectorRuta CSS propuesta para hacer click, escribir o localizar un control durante una automatización.
WorkflowReceta YAML con pasos repetibles: abrir página, esperar UI5, click, escribir, pulsar Enter y guardar snapshot.

Elección rápida

Qué modo debes usar

Sesión Fiori viva

Usa este modo cuando tengas acceso real a SAP/Fiori. Es el más completo porque detecta UI5, modelos y estado dinámico.

Recomendado para automatización profesional
HTML guardado

Úsalo para aprender, documentar una pantalla o analizar evidencias sin conectarte al entorno SAP.

Bueno para pruebas offline
Regla práctica: si puedes abrir la app Fiori real, usa CDP. Si solo tienes un fichero HTML, usa análisis offline.

Ruta guiada

Cómo trabajar paso a paso

01
Arranca el backend Rust

Ejecuta cargo run -- --config config/local.toml serve y abre http://127.0.0.1:7820.

02
Abre Chrome DevTools Protocol

Normalmente no tienes que hacer nada: la app intenta lanzar Chrome sola. Si falla, ejecuta manualmente google-chrome --remote-debugging-port=9222 --user-data-dir=./.browser-profile-cdp.

03
Captura la pantalla Fiori

Pega la URL Fiori. Si hay login corporativo, entra manualmente y espera a que la página quede cargada.

04
Lee el resumen

Revisa calidad, número de controles, acciones candidatas, endpoints OData y riesgos.

05
Elige acciones fiables

Prioriza selectores basados en IDs estables, textos claros y controles visibles. Evita coordenadas de pantalla.

06
Genera y prueba un workflow

Edita el YAML, valida cada paso y ejecútalo de forma completa o incremental para revisar evidencias.

Buenas prácticas

Automatización segura y mantenible

  • Trabaja primero en entorno de desarrollo o calidad, no en producción.
  • No guardes contraseñas en workflows, código fuente ni capturas JSON.
  • Usa usuarios técnicos o roles controlados cuando la política SAP lo permita.
  • Prefiere OData/API para operaciones críticas y UI solo para descubrimiento o validación visual.
  • Documenta cada workflow con objetivo, pantalla, transacción equivalente y riesgos.
  • Versiona los workflows en Git para saber cuándo y por qué cambian.

Solución de problemas

Errores frecuentes y cómo resolverlos

No captura nada o dice que CDP no está disponible

Abre otra terminal y ejecuta google-chrome --remote-debugging-port=9222 --user-data-dir=./.browser-profile-cdp. Luego comprueba curl http://127.0.0.1:9222/json/version.

La página queda en login corporativo o SSO

Configura más tiempo de espera en config/local.toml, parámetro manual_login_wait_secs, para poder autenticarte manualmente.

No detecta SAPUI5

Puede que la pantalla esté dentro de un iframe, no sea Fiori/SAPUI5, o todavía no haya terminado de cargar. Prueba a esperar más tiempo o analizar desde una URL más interna.

Los selectores parecen muy largos o inestables

Prioriza candidatos como [id$='--nombreControl'], textos visibles, roles ARIA y controles con alta confianza. Evita selectores con índices frágiles.

La automatización falla después de cambiar de pantalla

Añade pasos de espera, captura un nuevo snapshot después de cada navegación y usa selectores del estado final, no del estado inicial.
